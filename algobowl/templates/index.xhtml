How it Works

Students are placed in groups and receive an NP-hard problem. They are asked to design a good heuristic to the problem, create an input to the problem, and a program which, given an input and output, will verify if the output is a valid solution to the input.

Stage 1: Input Upload

Groups upload their inputs to the problem and select a team name.

Stage 2: Output Upload

Groups run all inputs on their solution program and upload their outputs to the site.

Stage 3: Verification

Groups verify that each solution to their input from all other groups is valid.

If one group rejects another group's output, the rejected group will have 24-hours to either upload a new output, or protest the rejection. If the new output is accepted by the professor, they will get to keep it, but it will cost a penalty point.

After this 24-hour period, the following stages open:

Stage 4: Open Verification

All outputs for all inputs become available. Some teams may wish to run their verifier on their verifier to ensure there were no false-positive verifications.

Stage 5: Evaluation

Groups submit evaluations of their teammates effort contributions to be considered in their grade.
